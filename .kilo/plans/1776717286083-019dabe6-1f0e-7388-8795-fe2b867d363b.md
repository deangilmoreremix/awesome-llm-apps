# Plan: Fix App Launch Links in VideoRemix AI Hub

## User Preference Confirmed
- User wants multi-page hub experience
- No individual app deployments
- Focus on local running with reliable launches

## Current Problem
- Launch buttons in the hosted hub show run commands but don't open apps
- Apps exist in the repository but aren't accessible in Streamlit Cloud environment
- Users expect clicking "Launch App" to open the app directly

## Root Cause Analysis
- Hub deployed from `videoremix-ai-hub/` folder only
- Individual app source files not available in hosted container
- Subprocess launching impossible in Streamlit Cloud
- No deployed URLs for individual apps

## Selected Solution: Local-Only Multi-Page Hub

**Rationale:**
- Maintains multi-page navigation structure
- Enables full launch functionality via subprocess
- Avoids complexity of individual deployments
- Provides complete local experience

**Pros:**
- All apps work via subprocess launching
- Multi-page hub with sidebar navigation
- No deployment complexity or costs
- Full control over app execution

**Cons:**
- Users must clone and run locally
- No hosted discovery experience
- Requires local Python/Streamlit setup

## Implementation Steps

### 1. Update Hub for Local-First Experience
- Modify launcher to prioritize subprocess launching
- Remove hosted-only fallbacks
- Add clear local setup instructions

### 2. Enhance Multi-Page Structure
- Ensure all hub pages work correctly
- Add app status indicators
- Improve navigation UX

### 3. Documentation Updates
- Update README for local usage
- Add setup instructions
- Provide troubleshooting for launches

### 4. Launch Reliability
- Test subprocess launching for all apps
- Handle edge cases (missing files, permissions)
- Add launch status feedback

### 5. User Experience
- Clear instructions for getting started
- One-click local setup scripts
- Helpful error messages

## Success Criteria
- Hub runs locally as multi-page app
- Clicking "Launch App" opens app in new tab/window
- All apps launchable from hub interface
- Clear setup process for new users
- No dependency on hosted deployments
